Структура репозитория.

Структура:
 /
    /trunk
    /tags/
        /0.0.1
        /0.0.2
        ...
    /branches/
        /0.0.1
        /0.0.2
        ...

/trunk — основная ветка разработки проекта. В нее вносятся все изменения и исправления ошибок.

/tags содержит релизы проекта. Именно из поддиректорий директории /tags исходный код выкладывается на рабочие сервера. 

/branches необходима для упрощения внесения больших изменений в код проекта. В ней хранятся ветви разработки. Если человек разрабатывает большую фичу, то он должен создать себе бранч и время от времени синхронизировать его с trunk. По окончании разработки фичи этот бранч сливается с транком и удаляется.

Итак, /trunk — основная ветвь разработки системы. Релизы создаются на основе содержимого директории /trunk. Создавать релизы на основе бранчей в данном случае не рекомендуется.
Создание структуры (способ 1)
 svn co http://svn.loc .

svn mkdir /home/svn/some_repo/tags
svn mkdir /home/svn/some_repo/branches
svn mkdir /home/svn/some_repo/trunk 

 svn ci -m "import structure"

 По очереди создаем папки и делаем комит.
Создание структуры (способ 2)
 svn import /../svn_structure http://svn.loc -m"import structure"
 Создать временную директорию со структурой и закомитить ее.
Импортирование файлов в репозиторий.

svn import /../project  http://svn.example.com/repos/calc/trunk -m "import"

 /../project путь к проекту который будет импортирован в репозиторий
Если импотр проекта был не успешным для повторного его проведения необходимо сделать чекаут удалить все файлы и папки потом сделать коммит изминений после чего повторный импорт проекта
Экспорт файлов из репозитория svn export [URL репозитория или файла/директории в нем] [target]
 Получение проекта или его части без служебных папок. Если не указать target то експорт будет включать и папку которую експортирем (trunk например)
