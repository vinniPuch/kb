Статусы файлов / свойства файлов

Статусы файлов
элемент был запланирован для добавления в репозиторий (“Added”);
 A
элемент находится в состоянии конфликта (“Conflict”);
 C
элемент запланирован для удаления из репозитория (“Deleted”);
 D
элемент был изменен (“Modified”);
 M
элемент будет заменен (“Replaced”); R
элемент не имеет версий;
 X
файл обновлен (“Updated”); U
файл совмещен, например, командами merge или update;
 G
этот элемент не будет добавлен в коммит и не имеет версий (например, только что созданный файл);
 ?
элемент находится под контролем версий, но отсутствует или поврежден. Элемент может отсутствовать, если был удален с помощью не-Subversion команды;
 !
элемент в хранилище и в рабочем каталоге — разных типов;
 ~
элемент известен системе и игнорируется (добавлен пользователем в список игнорирования).
 I
Cвойства файлов
пометить файл как исполняемый;
 svn:executable
определяет MIME-тип файла; svn:mime-type
определяет маску для каталога, по которой будут игнорироваться файлы;
 svn:ignore
имена ключевых слов, которые клиент должен разыменовать в файле; svn:keywords
указывает клиенту механизм обработки символов конца строк в рабочей копии; svn:eol-style
определяет многострочный список других частей и URL; svn:externals
указывает на то, что он не является обычным файлом; svn:special
наличие этого свойства у файла указывает клиенту пометить его в рабочей копии доступным только для чтения; svn:needs-lock
указывает автора редакции;
 svn:author
дата создания в ISO формате; svn:date
содержит сообщение к редакции;
 svn:log
если установлено, то эта редакция создана автоматически.
 svn:autoversioned
Пример изминения свойвств  файла(в данном случае папки) svn propedit svn:ignore .
