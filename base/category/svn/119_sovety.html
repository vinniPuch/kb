Советы

1) Решение проблеми с паролями( точнее
менеджером паролей операционной истемы.
Возможно, вы столкнетесь с таким 
сообщением: Password for ‘(null)’ GNOME keyring:, и система не разрешит вам   ввести имя пользователя для доступа к репозиторию. Втаком случае откройте файл ~/.subversion/config в         текстовом редакторе раскомментируйте строку “password-stores” в секции auth] и удалите все значения. Строка должна выглядеть так:
ФАЙЛ:
~/.SUBVERSION/CONFIG password-stores = Этими настройками вы отключите менеджеры паролей, встроенные в оболочку вашей операционной истемы,
которые и мешают правильной работе SVN.
2) Для перемещения файлов всегда пользоваться средствами svn.
svn move style.css misc/style.css
3) При работе в команде необходимо       регулярно делать svn up что в какой-то мере убережот от конфликтов.
svn up
4) Ветки предназанчены для работы с
проектом и развитием его в то время как теги снимки проекта в определенный момент времени.
5) Как хранить файл конфигурации     приложения который разный для разных
машин.
В репозиторий добавить файл-шаблон конфигурации. Сам же файл конфигурации добавить в игнор. Когда пользователю нужен файл нфигурации он копирует шаблон в рабочий файл конфигурации и изменяет его под себя. Таким образом локальный           конфиг не попадет в репозиторий а от туда и на боевой сервер.
