Просмотр индексированных и неиндексированных изменений и их фиксация

Что конкретно поменялось, а не только какие файлы были изменены  git diff Результат показывает ещё не проиндексированные изменения.

Отобразит, что же вы изменили, но пока не проиндексировали.
Просмотр уже проиндексированных изминений git diff --staged Эта команда сравнивает ваши индексированные изменения с последним коммитом
Фиксация изминений, после вызова команды откроеться редактор для создания комментария к коммиту. git commit

# Please enter the commit message for your changes. Lines starting
# with '#' will be ignored, and an empty message aborts the commit.
# On branch master
# Changes to be committed:
#   (use "git reset HEAD <file>..." to unstage)
#
#       new file:   README
#       modified:   benchmarks.rb
~
~
~
".git/COMMIT_EDITMSG" 10L, 283C

Для ещё более подробного напоминания, что же именно вы поменяли, можете передать аргумент -v в команду git commit. Это приведёт к тому, что в комментарий будет также помещена дельта/diff изменений, таким образом вы сможете точно увидеть всё, что сделано.
Короткий способ чтобы сразу указать комментарий.
git commit -m "Story 182: Fix benchmarks for speed"
Если использовать ключ -a то все файлы автоматически попаду в индекс, что упрощает комит потому что нет необходимости делать git add для файлов которые мы хотим добавить в индекс.
git commit -a -m 'added new benchmarks'
